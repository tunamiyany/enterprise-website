# 安装说明

## 当前安装情况

你正在运行 `快速安装.bat`，npm 正在下载 16 个依赖包。

### 正在发生什么？

1. **配置淘宝镜像** ✓ 已完成
2. **下载依赖包** ⏳ 进行中
   - 显示的 `deprecated` 警告是正常的，不影响使用
   - npm 正在后台下载和编译包
   - 预计还需要 3-5 分钟

### 为什么这么慢？

主要耗时的包：
- **next** (~50MB) - Next.js 框架
- **@prisma/client** (~30MB) - 数据库客户端
- **typescript** (~20MB) - TypeScript 编译器
- **tailwindcss** (~15MB) - CSS 框架

这些包需要下载和编译，首次安装确实需要时间。

## 三种安装方案

### 方案1: 继续等待当前安装 ⏳
- **时间**: 还需 3-5 分钟
- **包含**: 16 个包（核心功能 + 数据库 + 多语言）
- **建议**: 如果已经等了几分钟，建议继续等待

### 方案2: 极速安装（推荐） ⚡
- **时间**: 1-2 分钟
- **包含**: 13 个包（仅核心功能）
- **操作**: 
  1. 按 Ctrl+C 停止当前安装
  2. 运行 `极速安装.bat`
- **限制**: 暂时没有数据库和多语言功能

### 方案3: 手动安装
```bash
# 1. 停止当前安装 (Ctrl+C)
# 2. 清理
rd /s /q node_modules
del package-lock.json

# 3. 使用 cnpm 安装（更快）
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install
```

## 常见问题

### Q: 为什么有这么多 deprecated 警告？
A: 这些是依赖包的版本警告，不影响使用。npm 生态中很常见。

### Q: 可以跳过某些包吗？
A: 可以，使用 `极速安装.bat` 只安装核心包。

### Q: 下次安装还会这么慢吗？
A: 不会。npm 会缓存已下载的包，下次安装会快很多。

### Q: 安装失败怎么办？
A: 
1. 删除 `node_modules` 文件夹
2. 删除 `package-lock.json` 文件
3. 重新运行安装脚本
